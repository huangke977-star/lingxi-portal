# Redis 缓存管理设计

## 目标

为 HLOVET 超级管理员提供内置 Redis 管理页面，在不暴露 Redis 公网端口和任意命令控制台的前提下，查看运行状态、扫描缓存键、检查脱敏键值，并执行具有业务一致性的安全操作。

## 权限

- 页面入口：头像菜单中的“缓存管理”。
- 页面路由：`/admin/cache`。
- API 路由：`/api/admin/cache/*`。
- 前后端都要求 `isSuperAdmin=true`。
- 管理员角色即使等级为 90 也不能访问。

## 功能

- Redis 版本、运行时间、客户端数、键总数、内存、命中率、过期和淘汰统计。
- 使用 `SCAN` 游标按键名和类型分批查询，每批最多 100 个键。
- 查看 String、Set、Hash、List、ZSet、Stream 内容，单次最多展示 200 个元素或 64 KB 字符串。
- 自动格式化 JSON，并对密码、令牌、密钥、Cookie、认证字段递归脱敏。
- `tokenHash` 仅显示少量首尾字符，真实 Refresh Token 永不进入管理页面。
- 支持最多 20 个键的批量处理。
- 删除 `refresh_token:*` 时同步移除 `user_sessions:*` 索引。
- 删除 `user_sessions:*` 时同步撤销该用户全部刷新会话。
- 删除 `login_fail:*` 时清除对应登录失败计数。
- 普通业务缓存允许修改 TTL；认证缓存 TTL 禁止人工修改。

## 安全限制

- 不提供 `KEYS *`、`FLUSHDB`、`FLUSHALL` 或任意 Redis 命令执行入口。
- Redis 继续仅存在于 Docker 内部网络，不开放 `6379` 公网端口。
- 键值按需读取，不进行高频全量轮询。
- 概览每 30 秒刷新，键列表仅在筛选、翻批或手动刷新时读取。
- 删除和会话撤销操作必须由前端二次确认。

## 验证

- 普通用户和管理员访问返回 403。
- 超级管理员可以读取概览、键列表和脱敏详情。
- 敏感字段不能在响应中出现原值。
- 会话删除同时维护令牌键和用户会话集合的一致性。
- 认证缓存 TTL 修改返回 400，业务缓存 TTL 可更新。
- API 测试、前后端 lint 和生产构建全部通过。
