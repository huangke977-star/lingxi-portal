# 灵犀门户 UI 重构设计

日期：2026-07-10

## 目标

把当前偏 demo 的大标题页面重构为现代工作台风格，让公开入口、登录工作台、导航、工具箱和管理后台看起来像同一个长期可用的门户系统。

## 视觉方向

视觉主题：冷静、清爽、偏开发者工作台；以浅色中性背景、细边框、清晰层级和少量蓝色/青绿色状态色为主。

内容规划：

- 全站使用统一 app shell：侧边栏导航、右侧工作区、移动端自动变为顶部导航。
- 首页改成门户总览，展示账号入口、导航入口、工具箱入口和 API 状态。
- 工作台展示当前身份、角色境界和下一步可用区域。
- 导航页和工具箱页使用入口列表，而不是空白占位。
- 管理后台保持表格优先，提高密度和可读性。

交互主题：

- 导航和入口项使用轻微位移、边框和背景过渡。
- 状态徽章、角色徽章使用稳定颜色，不做花哨动画。
- 弹窗和表格保留现有交互逻辑，只优化视觉层级。

## 范围

本阶段包含：

- 重构 `RootLayout` 为统一工作台外壳。
- 重写全局 CSS 设计变量、布局、按钮、表单、表格、弹窗和入口列表样式。
- 改造首页、工作台、导航页、工具箱页、登录页、注册页和管理后台的结构与文案。
- 保留当前后端 API、认证逻辑和管理功能。
- 暂时使用静态导航/工具示例数据撑起布局，后续接数据库。

本阶段不包含：

- 引入 Tailwind、shadcn/ui 或大型 UI 依赖。
- 新增数据库表。
- 新增导航/工具 CRUD。
- 新增图标库。
- 深色模式。

## 设计规则

- 主界面不使用大 hero，不做营销页。
- 页面标题使用工作台尺度，不使用夸张展示字号。
- 卡片只用于入口项、状态块、表单和弹窗，不做层层嵌套。
- 角色境界只作为轻量识别，例如角色徽章和说明文案，不做古风装饰。
- 移动端必须避免文本溢出，侧边栏必须变为可横向扫描的顶部导航。
- 保持可部署性：只改前端页面和 CSS，不影响 API 容器配置。

## 验收标准

- 首页、工作台、导航、工具箱、登录、注册和管理后台形成统一视觉系统。
- 公开页面和后台页面都能在移动端正常阅读。
- 管理后台现有功能不变，包括角色修改、启停、修改密码。
- 页面不再出现大面积空白占位。
- `pnpm --filter @lingxi/web lint` 通过。
- `pnpm --filter @lingxi/web build` 通过。
- `pnpm build` 通过。
