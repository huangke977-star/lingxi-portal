# HLOVET 全站背景管理设计

## 目标

- 所有主题共用一张由超级管理员选择的全站背景图。
- 普通用户的主题偏好只控制配色和卡片透明度。
- 超级管理员可以上传、查看、切换和永久删除图片。
- 容器重建后保留已上传文件。

## 权限

- `GET /backgrounds/active` 和 `GET /backgrounds/files/:storedName` 公开可读。
- 图片列表、上传、激活和删除接口同时使用 `JwtAuthGuard` 与 `SuperAdminGuard`。
- 管理页面位于 `/admin/backgrounds`，不在公开门户中提供入口。

## 数据与文件

- MySQL 的 `background_images` 表保存原始文件名、随机文件名、MIME 类型、大小、激活状态、上传者和时间。
- 文件写入 `BACKGROUND_UPLOAD_DIR`，Docker 默认挂载命名卷 `background_uploads` 到 `/app/uploads/backgrounds`。
- 上传文件名由 UUID 生成，不接受客户端文件路径。
- 接受 JPEG、PNG、WebP、AVIF，单张最大 10 MB。
- 服务端同时校验 MIME、扩展名和二进制文件签名。

## 状态行为

- 上传后图片进入图片库，但不会自动替换当前背景。
- 激活图片时在数据库事务中先清除旧激活状态，再设置新图片。
- 删除当前图片后，不自动选择其他上传图片，门户恢复内置默认背景。
- 删除操作先删除真实文件，再删除数据库记录；磁盘文件不存在时仍允许清理遗留记录。

## 前端行为

- `ThemeController` 加载公开的当前背景接口，并设置全局 CSS 变量 `--portal-bg-image`。
- 所有主题不再覆盖背景图片变量。
- 管理员切换或删除当前背景时会触发页面内背景刷新事件，无需重新登录。
- 管理页面展示缩略图、文件大小、上传时间、上传者和当前使用状态。

## 验证

- 普通用户无法访问管理接口。
- 伪造 MIME 的非图片文件被拒绝。
- 连续切换图片后只有一张处于激活状态。
- 删除图片后数据库记录和磁盘文件同时消失。
- 删除当前图片后公开接口返回空值，门户使用内置默认背景。
